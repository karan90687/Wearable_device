# MAX30102 Driver Documentation

This document explains the ESP32 MAX30102 driver in `sender_node/components/max30102`, based on `max30102.c` implementation comments.

## 1. Overview

- Device: MAX30102 (Pulse oximetry and heart-rate sensor)
- I2C Address macro: `MAX30102_I2C_ADDR` (from `max30102.h`)
- I2C bus: `I2C_NUM_0`
- ESP-IDF API used: `i2c_master_write_to_device`, `i2c_master_write_read_device`
- Logging tag: `max30102`

## 2. Public API

### `esp_err_t max30102_check_device(void)`
- Reads register `0xFF` (PART_ID)
- Expected `part_id = 0x15`
- Returns `ESP_OK` if matched, `ESP_FAIL` otherwise

### `esp_err_t max30102_init(void)`
Initialization sequence:
1. `max30102_check_device()`
2. `reset_registers()`
3. `fifo_configure(0x50)` (4 sample average, rollover, A_FULL=0)
4. `spo2_configure(0x01, 0x03, 0x03)` (4096 nA, 100Hz, 411µs)
5. `set_led_current(0x24, 0x24)` (RED/IR ~7mA)
6. `set_mode_spo2()` (SpO2 mode)
7. `reset_fifo()`
- Logs success

## 3. Internal helper functions

### `max30102_read_reg(uint8_t reg, uint8_t *data, size_t len)`
- Reads from a register using I2C repeated-start flow.

### `max30102_write_reg(uint8_t reg, uint8_t data)`
- Writes a register value with I2C write.

## 4. Configuration functions

### `fifo_configure(uint8_t samples)`
- Writes `0x08` FIFO_CONFIG:
  - bits 7:6 SMP_AVE
  - bit 5 ROLLOVER_EN
  - bits 3:0 A_FULL
- Example values:
  - `0x78`: 8 sample average, rollover on
  - `0x50`: 4 sample average, rollover on (used)

### `spo2_configure(uint8_t adc, uint8_t sr, uint8_t pw)`
- Writes `0x0A` MODE_CONFIG:
  - bits 6:5 ADC_RGE (2048/4096/8192/16384 nA)
  - bits 4:2 SR (sample rate)
  - bits 1:0 LED_PW (pulse width 69/118/215/411 us)
- Combined value: `(adc << 5) | (sr << 2) | pw`

### `set_led_current(uint8_t red_current, uint8_t ir_current)`
- Writes `0x0C` LED1_PA (RED) and `0x0D` LED2_PA (IR)
- Common settings:
  - `0x24` = ~7 mA
  - `0x3F` = ~12 mA

### `set_mode_spo2(void)`, `set_mode_heart_rate(void)`, `set_mode_multi_led(void)`
- Writes `0x09` MODE_CONFIG:
  - SpO2: `0x03`
  - Heart Rate: `0x02`
  - Multi-LED: `0x07`
- Maintains SHDN = 0, RESET = 0

### `reset_registers(void)`
- Reads `0x09`, sets RESET bit (bit 6), writes back, waits 10ms.

### `reset_fifo(void)`
- Writes `0x00` to:
  - `0x04` FIFO_WR_PTR
  - `0x05` OVF_COUNTER
  - `0x06` FIFO_RD_PTR

## 5. Notes and caveats

- The driver currently uses literal constants and does not expose user-level config functions.
- In `max30102_init()`, the variable `reti` is used before declaration; change to `esp_err_t reti`.
- In `spo2_configure()`, code uses `adc_range`, `sample_rate`, `pulse_width` but parameter names are `adc`, `sr`, `pw`; fix to avoid compile error.

## 6. Future enhancements
